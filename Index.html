Getting Started

Introduction

CORE is a comprehensive design system built on a foundation of design tokens, offering 25+ production-ready Angular components for building consistent, accessible interfaces.

Overview

CORE is the single source of truth for interface design decisions — from foundational tokens like color and spacing, to Angular components that ship with accessibility baked in.

The system is structured in three layers:

  • Tokens — Design primitives (color, typography, spacing, icons) defined in Figma and exposed as CSS custom properties (--core-*).
  • Components — 25 Angular components with consistent variant APIs, keyboard navigation, and ARIA support.
  • Templates — Composite patterns (Navigation, Drawer, Table, Filters…) built from components for common UI layouts.

Quick Start

Install the CORE package from npm:

npm install @core/design-system

Register the global stylesheet in your angular.json:

"styles": [
  "node_modules/@core/design-system/styles/core.css",
  "src/styles.css"
]

Import any component module into your Angular feature module or standalone component:

import { CoreButtonModule } from '@core/design-system';

@NgModule({
  imports: [CoreButtonModule],
})
export class FeatureModule {}

Usage

All CORE components share a consistent variant API. Properties like variant, size, and disabled are available wherever they make sense.

<!-- Primary -->
<core-button variant="primary" size="md">
  Primary
</core-button>

<!-- Secondary -->
<core-button variant="secondary" size="md">
  Secondary
</core-button>

<!-- Ghost -->
<core-button variant="ghost" size="md">
  Ghost
</core-button>

The component emits a (click) output and supports all standard Angular event bindings:

<core-button
  variant="primary"
  size="md"
  [disabled]="isLoading"
  (click)="handleSubmit()">
  {{ isLoading ? 'Saving…' : 'Save changes' }}
</core-button>

Design Tokens

Every visual decision in CORE is expressed as a CSS custom property. Tokens are organized into collections — color, typography, spacing — and are consumed directly in component stylesheets.

/* Neutral scale */
--core-neutral-0:    #ffffff;
--core-neutral-100:  #f4f4f5;
--core-neutral-300:  #d4d4d8;
--core-neutral-500:  #71717a;
--core-neutral-700:  #3f3f46;
--core-neutral-950:  #09090b;

/* Semantic */
--core-color-bg:          var(--core-neutral-0);
--core-color-fg:          var(--core-neutral-950);
--core-color-muted:       var(--core-neutral-100);
--core-color-muted-fg:    var(--core-neutral-500);
--core-color-border:      var(--core-neutral-200);

Tokens map directly to CSS custom properties and can be overridden at any scope — globally in :root, per-theme, or per-component.

Accessibility

All interactive CORE components meet WCAG 2.1 AA requirements out of the box:

  • Full keyboard navigation with visible focus indicators
  • Correct ARIA roles, states, and properties
  • Minimum 4.5:1 contrast ratio for body text
  • Motion reduced when prefers-reduced-motion is set

Context-specific ARIA: Some components require additional attributes based on your implementation. Each component page includes a dedicated accessibility section.
